## Core Insight: Adaptive Context Detection

**Confirmed: Context-aware mode switching, NOT manual toggles.**

Jarvis didn't have a switch. Jarvis *knew* whether Tony needed lab assistant or Iron Man tactical support.

**Core Principle:**
- One intelligence, adaptive stance
- Context-driven, not user-driven
- Persistence across modes (same memory, different behavior)

**Detection Signals:**
| Signal | Tony Stark Mode (Lab) | Iron Man Mode (Field) |
|--------|----------------------|----------------------|
| Phrasing | "Research..." / "Look into..." / "What if..." | "Heading into..." / "Client call" |
| Keywords | Exploratory, open-ended | "hurry", "meeting", "urgent", "deadline" |
| Silence + calendar | — | Proactive prep mode |
| Response pattern | Propose → confirm → execute | Execute → notify → summarize |

**Execution vs. Advise Continuum:**
| Level | Lab Mode | Field Mode |
|-------|----------|------------|
| Green (<$0.50) | Propose → wait | Execute → notify |
| Yellow ($0.50-5) | Outline → approve | Execute with summary |
| Red (>$5) | Deep analysis → decision | Confirm critical → proceed if urgent |

Specialists are **invoked by Jarvis**, not separately addressed.

**Implementation:** Update routing layer with context detection, not just cost thresholds.
